The admin of Acme Inc. creates the authentication widget on the CyberArk Identity portal and downloads HTML from the portal to embed the signup form.

The signup widget internally uses the “/Signup” API of CyberArk Identity to signup and create the users.
The widget’s fields and look and feel dynamically change based on the widget ID.
The signup form supports creating users using a bearer token (or) captcha. You can use the widget alongside Identity verification to prevent identity theft and consent management.
You can integrate the widget with authentication and authorization protocols like OIDC/SAML to generate tokens for the user if auto-login is enabled.

The admin of Acme Inc. creates the authentication widget on the CyberArk Identity portal and downloads the HTML from the portal to embed the login form.

The login form prompts the user with the MFA as per the user or app policy.
The widget’s fields and look and feel dynamically change based on the widget ID.
You can integrate the widget with authentication and authorization protocols like OIDC/SAML. By default, the widget returns a one-time authentication token.
